# Support

## How to get help

**Before opening an issue**, check:

1. [README.md](README.md) — quick start, flags, configuration, API reference
2. [TOOLS.md](TOOLS.md) — tool inventory and per-tool notes
3. [Existing issues](../../issues) — your problem may already be reported or resolved

## Bug reports

Use the [Bug Report](../../issues/new?template=bug_report.md) issue template.
Include your platform, shell, full terminal output, and any `INSTALL_RECEIPT.txt` contents.

## Feature requests

Use the [Feature Request](../../issues/new?template=feature_request.md) issue template.

## Questions and discussion

For general questions, use [GitHub Discussions](../../discussions).
Issues are for bugs and feature requests — not for how-to questions.

## Security vulnerabilities
